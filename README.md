# Local Small Language Model v2 (LSLM)

## 概要

LSLM（Local Small Language Model v2）は、低計算資源環境で高速に動作しながら、人間の思考プロセスを近似することを目的とした軽量言語モデルです。

本プロジェクトは従来のLLMの「トークン生成モデル」という枠組みから離れ、「思考の発散と収束」による探索型アプローチを中核に据えています。

また、ローカル環境での自律学習・高速応答・CPU特化を前提として設計されています。

---

## コンセプト

LSLMは以下の思想に基づいています。

* 思考とは「処理の種類」ではなく「処理の順序」である
* 人間の思考は発散と収束の繰り返しで構成される
* 精度はモデルサイズではなく思考回数で制御できる
* 小型モデルでも設計次第で実用的な知的挙動が可能

---

## 特徴

* 思考フローを明示的に再現
* 発散と収束による意味探索
* 思考回数による精度制御
* CPU環境での高速動作を前提
* 外部LLMによる評価ベース強化学習
* 超軽量モデル（TinyLlama / Qwen系など）との統合前提

---

## 性能目標

* 動作環境：低スペックCPU（例：Intel Core i5-7200U）
* 応答速度：500ms以内
* 出力：1〜3文の短文応答（基本設計）

長文応答も可能だが、応答時間は増加してよい設計。

---

## 思考フロー

LSLMは以下の順序で処理を行います。

1. 入力の受け取り
2. 意図の推定
3. 内部状態との照合
4. 発散（候補生成）
5. 収束（評価・選択）
6. 出力生成

これらは独立したモジュールではなく、単一処理系の参照順序として実装されます。

---

## 思考単位（思考トークン）

LSLMにおける最小思考単位は以下の2種類です。

* 発散：関連単語・概念を広げる
* 収束：不要要素を削減し焦点を絞る

思考はこの2つの操作の反復で構成されます。

---

## モード構造

LSLMは思考回数に応じて2つのモードを持ちます。

* 高速モード：発散・収束回数を最小化
* 推論モード：思考回数を増やして精度向上

構造は同一であり、違いは思考回数のみです。

---

## 表現構造

* 単語単位で処理
* 各単語に多次元パラメータを付与
* パラメータ空間の距離で意味的関連性を表現

この空間上で近傍探索を行うことで発散が実現されます。

---

## 発散と収束の実装方針

### 発散

* 意味空間上の近傍探索
* 再帰的に候補を拡張

### 収束

* 意図・文脈・内部状態を基準に評価
* 不要な候補を削除

---

## 学習方式

LSLMは強化学習ベースで最適化されます。

1. 発散・収束を複数パターン試行
2. 各結果を外部評価モデルへ入力
3. 評価スコアを報酬として取得
4. 発散・収束パラメータを更新

---

## 評価モデル

外部評価にはオンラインLLMを使用します。

例：

* Gemini API
* OpenAI API（フォールバック）

評価対象：

* 発散の妥当性
* 収束の適切性
* 応答の自然さ
* 一貫性
* 思考効率

---

## 報酬設計

報酬は以下で構成されます。

報酬 = 応答品質 + 発散品質 + 収束品質 - 計算コスト

これにより「少ない思考回数で良い答えを出す」方向に最適化されます。

---

## システム構成（実装ベース）

本プロジェクトは以下の構成で動作します。

* ローカルLLM（Student）

  * CPU前提の超軽量モデル
  * TinyLlama / Qwen系など

* 教師モデル（Evaluator）

  * Gemini API
  * OpenAI API（フォールバック）

* 学習ループ

  * セッション単位で会話生成
  * 外部評価
  * データセット生成
  * 再学習

---

## 設計思想（重要）

LSLMは以下の点で従来のLLMと根本的に異なります。

| 項目   | 従来LLM    | LSLM   |
| ---- | -------- | ------ |
| 基本原理 | トークン生成   | 思考探索   |
| 精度向上 | モデルサイズ   | 思考回数   |
| 推論   | ブラックボックス | 明示的フロー |
| 計算資源 | GPU前提    | CPU前提  |
