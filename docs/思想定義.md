# philosophy.md

## Local Small Language Model v3 思想定義

本ドキュメントは、Local Small Language Model（LSLM）プロジェクトにおける v3 世代の思想を定義するものである。

本バージョンでは、従来の実装や構造を一度リセットし、設計思想そのものから再構築を行う。

---

## 1. 根本思想

LSLM v3 は「大規模言語モデルの縮小版」ではない。

むしろ以下を目指す。

> **人間の思考プロセスを模倣する軽量探索型言語システム**

これは「確率的に文章を生成するモデル」ではなく、

* 入力を受け取る
* 意味的に展開する
* 関連候補を比較する
* 最適な応答へ収束する

という一連の思考過程を持つシステムである。

---

## 2. LLM との差異

一般的な LLM は以下の特徴を持つ。

* 巨大パラメータ
* 確率分布に基づくトークン生成
* 文脈に対する統計的近似

LSLM v3 はこれとは明確に異なる。

### LSLM v3 の特徴

* 小規模・低計算資源で動作
* 明示的な探索プロセスを持つ
* 中間状態が可視化可能
* 思考ステップごとに評価が可能
* 辞書ベースの意味構造を持つ

つまり、

> **ブラックボックスではなく、ホワイトボックスな思考エンジン**

である。

---

## 3. 思考モデル

LSLM v3 の中核は「探索」である。

思考は次の2つの基本操作で構成される。

### 3.1 発散（Divergence）

入力から関連する概念・語を展開する。

* 意味的近傍探索
* 類義語・関連語の取得
* 文脈的連想

これは「思いつく」という行為に相当する。

### 3.2 収束（Convergence）

候補集合から最適なものを選び取る。

* 不要要素の除去
* 文脈との整合性評価
* 応答構造への適合

これは「考えをまとめる」という行為に相当する。

---

## 4. 思想の転換（v2 → v3）

v2 では以下の問題があった。

* 発散と収束が密結合していた
* 出力生成が収束処理に依存していた
* 中間状態の責務が曖昧だった
* 学習が一括報酬に依存していた

v3 ではこれを以下のように分解する。

### 分離方針

* 発散 = 意味想起（Semantic Recall）
* 収束 = スロット充填（Slot Filling）
* 文生成 = 表層化（Surface Realization）

これにより、

* 各段階の責務が明確になる
* 学習・評価を段階ごとに分離できる
* エラー原因の特定が容易になる

---

## 5. パイプライン思想

LSLM v3 の基本処理は次の流れを取る。

```text
Input
  ↓
Intent Planning
  ↓
Semantic Recall
  ↓
Slot Filling
  ↓
Surface Realization
  ↓
Response
```

### 5.1 Intent Planning

入力の意図を判定し、応答の構造を決定する。

例:

* 状態確認
* 応答
* 同意
* 質問返し

### 5.2 Semantic Recall

辞書を用いて関連語を想起する。

### 5.3 Slot Filling

必要な意味要素を埋める。

### 5.4 Surface Realization

自然文へ変換する。

---

## 6. 辞書中心設計

LSLM v3 は辞書を中心に構成される。

辞書は単なる単語リストではなく、以下を持つ。

* 意味ベクトル
* 文法属性
* 階層構造
* 関係性

この辞書が

* 発散の起点
* 収束の素材

となる。

つまり、

> **モデルの知識はパラメータではなく辞書にある**

---

## 7. 学習思想

LSLM v3 の学習は以下の特徴を持つ。

### 7.1 段階別学習

* 意図推定
* スロット充填
* 表層化

を分離して学習する。

### 7.2 外部評価の活用

* Gemini
* OpenAI
* ローカル LLM

などを評価器として使用する。

### 7.3 教師データの構造化

単なる input / output ではなく

* intent
* slots
* target

を含む。

---

## 8. 設計原則

### 8.1 可視性

すべての中間状態をログに出す。

### 8.2 分離

各処理を独立モジュールとして設計する。

### 8.3 軽量性

CPU 環境でも動作することを前提とする。

### 8.4 制御可能性

挙動をパラメータとルールで制御可能にする。

### 8.5 拡張性

辞書・評価器・生成ルールを後から差し替え可能にする。

---

## 9. ゴール

LSLM v3 の最終目標は以下である。

* 人間の思考に近い応答生成
* 小規模で高品質な対話
* 解釈可能な推論プロセス
* 学習可能な構造

そして最終的には

> **「考えることができる軽量言語エンジン」**

を実現する。

---

## 10. まとめ

LSLM v3 は、

* LLM の縮小版ではなく
* 探索型思考エンジンとして
* 辞書を基盤に構築される

新しいアプローチである。

その核心は、

> 発散と収束による思考の再現

にある。
