# LSLM v3.1.0a 仕様書

## 1. 概要

Local Small Language Model（LSLM）v3.1.0a は、従来の大規模言語モデルとは異なり、
**探索型思考プロセスを明示的に再現する軽量言語エンジン**である。

本バージョンでは、辞書基盤を中心とした構造を確立し、
今後の発散・収束型推論および強化学習の基盤となる設計が整備されている。

思想の核は以下にある。

* 発散（Divergence）による候補生成
* 収束（Convergence）による最適化
* 辞書ベースの意味空間
* 中間状態の完全可視化

（詳細思想は philosophy.md を参照）

---

## 2. システム全体構造

LSLM v3.1.0a は以下の層構造を持つ。

```
[辞書データ]
    ↓
[辞書I/O層]
    ↓
[推論・学習層（将来実装）]
```

現行バージョンでは特に **辞書基盤の確立** に焦点が当てられている。

---

## 3. コア思想

### 3.1 探索型思考モデル

LSLM は確率生成ではなく、以下の思考ループを持つ。

```
Input
 → Semantic Recall（発散）
 → Slot Filling（収束）
 → Surface Realization
 → Response
```

これは人間の「思いつき → 取捨選択 → 言語化」に対応する。

---

### 3.2 責務分離（最重要原則）

LSLM v3 では以下の3分離を厳守する。

| 区分  | 内容                                   |
| --- | ------------------------------------ |
| 辞書  | 知識（Axis / Grammar / Slot / Relation） |
| 実行時 | 状態（State / Policy）                   |
| 保存  | 記録（Log / Training Data / Evaluation） |

この分離により設計破綻を防ぐ。

（詳細: 知識分割設計）

---

## 4. 辞書設計

### 4.1 辞書の役割

辞書は単なる単語集合ではなく、

* 意味ベクトル（Axis）
* 文法制約（Grammar）
* 意味役割（Slot）
* 語間関係（Relation）

を持つ「知識グラフ」である。

### 4.2 データ構造

主要構造:

* meta
* entries
* indexes
* lexicon（階層）

辞書は JSON / LSD / LSDLX の3形式で扱われる。

---

### 4.3 バイナリ辞書

#### .lsd

* 圧縮形式
* 全展開ロード

#### .lsdx

* インデックス付き
* mmap による高速アクセス
* 大規模辞書向け（推奨形式）

---

## 5. 日本語処理設計

LSLM v3 では日本語を以下に分離する。

* 語幹（意味）
* 活用（状態変化）
* 助詞（関係）
* 表層（出力）

これにより、

* 意味と文法を完全分離
* 推論と表現を独立

が可能になる。

---

Slot Filling

Surface Realization

Scoring

現行バージョンでは、このパイプラインは仕様レベルで定義されている。

---

## 7. ロギング仕様

LSLM v3 は完全ログ駆動である。

### 特徴

* 全中間状態を記録
* JSONL形式で保存
* デバッグログと運用ログを分離
* 起動時に自動ローテート

ログは以下を含む。

* input
* intent
* 候補生成
* slot構造
* スコア内訳
* 最終出力

---

## 8. 学習設計（準備段階）

LSLM v3 の学習は以下を前提とする。

* 外部評価器（Gemini / OpenAI / Local）
* 教師データ（intent / slots / target）
* reward 分解

報酬は

* semantic整合
* slot充足
* grammar適合
* input保持

などに分解される。
